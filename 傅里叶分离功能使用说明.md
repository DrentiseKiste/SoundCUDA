# EchoSense - 傅里叶说话人分离功能使用说明

## 功能概述

傅里叶说话人分离功能已集成到 EchoSense GUI 应用中，使用户能够：
- 使用傅里叶分析技术分离音频中的多个说话人
- 支持三种不同的分离算法
- 自动估计说话人数量
- 可视化分离结果
- 导出分离后的音频和特征数据

## 使用方法

### 1. 启动应用

在项目根目录下运行：

```bash
python -m front.Audio_Processing_App
```

或者：

```bash
cd front
python Audio_Processing_App.py
```

### 2. 加载音频

1. 点击"音频导入"标签页
2. 选择音频文件（支持 .wav, .mp3, .flac, .ogg 格式）
3. 可以拖拽文件到界面或点击"选择文件"按钮

### 3. 使用傅里叶分离功能

1. 点击"傅里叶分离"标签页
2. 配置分离参数：
   - **分离方法**：
     - NMF（非负矩阵分解）- 推荐用于音乐和语音分离
     - ICA（独立成分分析）- 适用于盲源分离
     - 聚类方法 - 基于频谱特征的自动聚类
   - **说话人数量**：
     - 选择"自动估计"让系统自动检测
     - 或手动指定 2-5 个说话人
   - **FFT窗口大小**：
     - 1024 - 适合短时音频
     - 2048 - 默认值，平衡精度和速度
     - 4096 - 适合长时间音频
3. 点击"开始分离"按钮
4. 等待处理完成（进度条会显示处理进度）

### 4. 查看结果

分离完成后，结果会显示在界面中：

- **分离结果文本框**：
  - 检测到的说话人数量
  - 每个说话人的基频、能量、频谱质心等特征
  - 分离后的音频片段信息

- **可视化分析**：
  - 原始音频波形和频谱图
  - 每个分离出的说话人的波形和频谱图
  - 可以直观对比不同说话人的频谱特征

### 5. 保存结果

点击"保存结果"按钮：
1. 选择保存目录
2. 系统会自动保存：
   - 每个说话人的音频文件（speaker_1.wav, speaker_2.wav, ...）
   - 说话人特征数据（speaker_features.json）

## 技术原理

### 短时傅里叶变换 (STFT)

将音频信号分成多个短时段，对每个时段进行傅里叶变换，得到时频表示：

```
STFT(audio) → 频谱矩阵 (频率 × 时间)
```

### 分离算法

#### 1. NMF（非负矩阵分解）

将频谱矩阵分解为两个非负矩阵的乘积：
- 基矩阵：表示每个说话人的频谱特征
- 系数矩阵：表示每个时刻各说话人的活跃程度

**优点**：计算效率高，结果可解释性好

#### 2. ICA（独立成分分析）

在频域应用独立成分分析，分离统计独立的信号源：

**优点**：无需预先知道说话人数量

#### 3. 聚类方法

使用 K-means 聚类对频谱特征进行分组：

**优点**：可以自动估计说话人数量

### 特征提取

每个分离出的说话人都会提取以下特征：

- **基频 (F0)**：声音的基本频率，与音高相关
- **能量 (RMS)**：声音的响度
- **频谱质心**：频谱的"重心"位置
- **MFCC**：梅尔频率倒谱系数，描述音色特征
- **时长**：说话人说话的持续时间

## 适用场景

### 适合的场景：
- 多人对话录音
- 会议记录分离
- 语音数据集构建
- 音频编辑预处理

### 局限性和注意事项：
- 分离效果受音频质量影响
- 说话人声音特征差异不明显时效果可能较差
- 重叠语音（两人同时说话）分离效果有限
- 建议使用较高采样率（16kHz+）的音频

## 性能对比

| 方法 | 速度 | 质量 | 自动估计 | 适用场景 |
|------|------|------|----------|----------|
| NMF | 快 | 中高 | 否 | 实时处理、流式 |
| ICA | 中 | 中 | 否 | 通用分离 |
| 聚类 | 慢 | 高 | 是 | 离线分析、最佳效果 |

## 故障排除

### 问题1：导入模块失败

**解决方案**：
```bash
pip install -r requirements.txt
```

### 问题2：分离结果不理想

**建议**：
1. 尝试不同的分离方法
2. 调整 FFT 窗口大小
3. 如果知道说话人数量，手动指定
4. 使用更高质量的音频源

### 问题3：程序运行缓慢

**建议**：
1. 减小 FFT 窗口大小
2. 使用 NMF 方法（速度最快）
3. 缩短音频文件长度

### 问题4：检测到的说话人数量不正确

**建议**：
1. 使用"自动估计"选项，让系统使用肘部法则确定
2. 手动尝试不同的说话人数量
3. 检查音频是否包含足够的语音内容

## 示例工作流

### 示例1：分离两人对话

1. 加载包含两人对话的音频
2. 选择"聚类方法"
3. 选择"自动估计"或指定"2"
4. 点击"开始分离"
5. 查看结果，分析每个说话人的特征
6. 保存结果

### 示例2：提取单个说话人

1. 加载多人音频
2. 使用聚类方法分离
3. 根据基频特征识别目标说话人
4. 保存该说话人的音频
